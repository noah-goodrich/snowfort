### Step 2: Create `domain/rules/_iac.py` — shared helper module
- `QUERY_HISTORY` fetcher using server-side `REGEXP_LIKE` aggregation (cache key `("QUERY_HISTORY_IAC", 30)`)
- Named column constants for the aggregated result set
- Helper: `detect_iac_tools(tag_refs, qh_rows) -> dict[str, list[str]]` — returns detected tools with evidence
- Helper: `tag_coverage_by_database(tag_refs) -> dict[str, float]` — MANAGED_BY tag coverage per database

### Step 3: Create `domain/rules/iac_drift.py` — 4 rule classes
- **`IacToolDetectionCheck` (OPS_015):** Detect IaC tool presence via MANAGED_BY tags, query comments, and service
  account naming patterns. Report detected tools as INFORMATIONAL. Report "No IaC tools detected" as INFORMATIONAL
  recommendation.
- **`IacDriftIndicatorsCheck` (OPS_016):** Per detected tool, check for drift signals:
  - Terraform/tag-based: objects with MANAGED_BY tag modified by non-service-account user (via QUERY_HISTORY DDL
    by non-service user on tagged objects). Severity varies by tool.
  - dbt: tables/views in dbt-managed schemas modified by non-dbt user (QUERY_HISTORY).
  - Generic: objects without MANAGED_BY tag in databases with >50% tagged objects.
- **`DbtGrantTargetValidationCheck` (GOV_025):** Parse QUERY_HISTORY for dbt-emitted GRANT statements (identified
  by `dbt` comment pattern). Flag grants targeting business roles instead of functional roles.
- **`DbtSchemaOwnershipCheck` (GOV_026):** Check schema owners from INFORMATION_SCHEMA. Flag schemas owned by
  dbt service account instead of dedicated DBO role.

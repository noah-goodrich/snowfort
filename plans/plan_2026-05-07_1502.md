## Implementation Plan: Security Posture Rules (Article Gap Analysis)

### Context

**Explored:** Rule engine (`Rule` base class, `check_online()` pattern), existing security rules (SEC_001–SEC_029),
Trust Center checks (SEC_017 CIS scanner, SEC_021 unresolved findings), private link rules (SEC_014, SEC_022),
data exfiltration (SEC_013), rule registry, conventions/thresholds, test patterns (mock cursor, errno-2003 graceful
degradation), `__init__.py` exports.

**Key findings:**
- SEC_030+ are free. COST_047 is the next free cost ID.
- SEC_017 checks CIS scanner *enabled*; SEC_021 checks *unresolved HIGH/CRITICAL findings*. Neither checks all scanner
  packages status (Security Essentials + Threat Intelligence) or surfaces threat intel specifically.
- SEC_014 checks "any network policy exists"; SEC_022 checks "PrivateLink configured but not enforced". Neither
  measures the private-to-public login *ratio*.
- No login-anomaly detection exists (LOGIN_HISTORY brute-force patterns).
- No session policy check exists.
- No export-volume heuristic exists (SEC_013 checks params only).
- No rekeying check exists.

### Implementation Steps

#### Step 1: Add `SecurityPostureThresholds` to conventions.py

New frozen dataclass with configurable thresholds for the new rules:
- `brute_force_attempts_threshold: int = 5`
- `brute_force_window_days: int = 7`
- `private_link_ratio_threshold: float = 0.80`
- `large_export_rows_threshold: int = 1_000_000`
- `large_export_window_days: int = 7`
- `inactive_user_days: int = 90`

Add it as a field on `RuleThresholdConventions`.

#### Step 2: Create `domain/rules/security_posture.py` (new file, 7 rules)

| Rule ID | Class Name | Severity | What it checks |
|---------|-----------|----------|----------------|
| SEC_030 | `TrustCenterScannerStatusCheck` | HIGH | Any Trust Center scanner package disabled (Security Essentials, CIS, Threat Intel) |
| SEC_031 | `SessionPolicyCheck` | HIGH | No session policy in account (SHOW SESSION POLICIES) |
| SEC_032 | `BruteForceDetectionCheck` | HIGH | 5+ failed logins from same user in 7d (LOGIN_HISTORY) |
| SEC_033 | `PrivateLinkRatioCheck` | MEDIUM | Ratio of private-endpoint logins < 80% threshold (LOGIN_HISTORY) |
| SEC_034 | `LargeExportVolumeCheck` | MEDIUM | Any single user exported >1M rows in 7d (COPY_HISTORY) |
| SEC_035 | `PeriodicRekeyingCheck` | MEDIUM | Periodic rekeying not enabled (SHOW PARAMETERS) |
| SEC_036 | `ThreatIntelligenceFindingsCheck` | CRITICAL | Trust Center Threat Intelligence has open findings |

All rules follow the established pattern:
- Subclass `Rule`, implement `check_online()`
- Use `is_allowlisted_sf_error()` for errno 2003 → return `[]`
- Non-allowlisted errors → `raise RuleExecutionError(self.id, str(exc), cause=exc) from exc`
- Accept `conventions` kwarg for threshold-based rules

#### Step 3: Create `COST_047` in `domain/rules/cost.py`

| Rule ID | Class Name | Severity | What it checks |
|---------|-----------|----------|----------------|
| COST_047 | `InactiveUserLicenseImpactCheck` | LOW | Users with no login in 90d still holding active role grants |

Queries `SNOWFLAKE.ACCOUNT_USAGE.USERS` joined with `GRANTS_TO_USERS` to find inactive
users with active grants (excluding SERVICE type).

#### Step 4: Update `domain/rules/__init__.py`

Add imports from `security_posture` module and `COST_047` from `cost.py`. Add all 8 new
class names to `__all__`.

#### Step 5: Register rules in `infrastructure/rule_registry.py`

Add all 8 new rules to the `get_all_rules()` list under a new comment block:
```python
# Directive G — Security Posture (article gap analysis: SEC_030–036, COST_047)
```

Rules with thresholds get `conventions=conventions`; others get `telemetry=telemetry`.

#### Step 6: Create `tests/unit/test_security_posture.py`

For each of the 8 rules, test:
1. **Pass case** — clean data → `[]`
2. **Fail case** — bad data → correct violation(s) with expected message content
3. **Graceful degradation** — errno 2003 → `[]` (for Trust Center / LOGIN_HISTORY rules)
4. **Non-allowlisted error** — raises `RuleExecutionError`
5. **Rule ID assertion**

Follow the `_cursor()` / `_cursor_raising()` / `_not_found_error()` helper pattern from
`test_session6_rules.py`.

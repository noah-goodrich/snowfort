### Step 2: Schema DDL — `SNOWFORT.AUDIT` tables

Create `packages/snowfort-audit/sql/dashboard_schema.sql`:

```sql
CREATE DATABASE IF NOT EXISTS SNOWFORT;
CREATE SCHEMA IF NOT EXISTS SNOWFORT.AUDIT;

CREATE TABLE IF NOT EXISTS SNOWFORT.AUDIT.SCAN_METADATA (
    scan_id         VARCHAR(36)     NOT NULL,   -- UUID
    scanned_at      TIMESTAMP_NTZ   NOT NULL DEFAULT CURRENT_TIMESTAMP(),
    account_id      VARCHAR,
    compliance_score FLOAT,
    grade           VARCHAR(1),
    total_violations INT,
    critical_count  INT,
    high_count      INT,
    medium_count    INT,
    low_count       INT,
    pillar_scores   VARIANT,        -- {"Security": 85.2, "Cost": 72.1, ...}
    pillar_grades   VARIANT,        -- {"Security": "B", "Cost": "C", ...}
    billing_model   VARCHAR,
    reliable        BOOLEAN,
    total_rules     INT,
    errored_rules   INT,
    PRIMARY KEY (scan_id)
);

CREATE TABLE IF NOT EXISTS SNOWFORT.AUDIT.SCAN_VIOLATIONS (
    scan_id         VARCHAR(36)     NOT NULL,
    rule_id         VARCHAR         NOT NULL,
    resource_name   VARCHAR,
    message         VARCHAR,
    severity        VARCHAR,        -- CRITICAL/HIGH/MEDIUM/LOW
    pillar          VARCHAR,
    category        VARCHAR,        -- ACTIONABLE/EXPECTED/INFORMATIONAL
    remediation_key VARCHAR,
    rationale       VARCHAR,
    quick_win       BOOLEAN,
    FOREIGN KEY (scan_id) REFERENCES SNOWFORT.AUDIT.SCAN_METADATA(scan_id)
);
```

### Step 3: Persist use case — `use_cases/persist_scan.py`

New module with a `PersistScanUseCase` class:
- Takes a gateway (cursor) and `AuditResult`
- Generates UUID scan_id
- Ensures `SNOWFORT.AUDIT` schema exists (runs DDL IF NOT EXISTS)
- INSERTs one row to `SCAN_METADATA` with scorecard fields
- Batch-INSERTs violations to `SCAN_VIOLATIONS` with enrichment fields
- Returns scan_id for confirmation messaging

### Step 4: CLI `--persist` flag on `snowfort audit scan`

Add `--persist` flag to the scan command:
- After scan completes and `AuditResult` is built, if `--persist` is set:
  - Use the existing gateway connection
  - Call `PersistScanUseCase.execute(gateway, result, rules)`
  - Print: "Scan persisted to SNOWFORT.AUDIT (scan_id: {uuid})"
- If SNOWFORT DB doesn't exist, the use case creates it (requires appropriate role — warn if insufficient privileges)

### Step 5: Multi-page Streamlit app — `packages/snowfort-audit/streamlit/`

New directory `packages/snowfort-audit/streamlit/` (separate from the Python package, as SiS apps are deployed from a stage, not imported as modules):

```
streamlit/
├── snowfort_app.py          # Main entry point (page config + navigation)
├── pages/
│   ├── 1_Dashboard.py       # Scorecard KPIs, pillar charts, severity donut
│   ├── 2_Explorer.py        # Drill-down: filter → click → rationale/remediation
│   └── 3_Trends.py          # Score over time, regression detection, scan comparison
└── environment.yml          # Python deps (plotly, pandas — streamlit is provided by SiS)
```

**Page 1 — Dashboard:**
- Reads latest scan from `SNOWFORT.AUDIT.SCAN_METADATA`
- 4 KPI metrics (score, total, critical, high) with delta vs previous scan
- Pillar score bar chart + severity donut (Plotly)
- Score trend sparkline (last 10 scans inline)

**Page 2 — Explorer:**
- Sidebar filters: pillar multiselect, severity multiselect, rule_id text search
- Violations table from `SNOWFORT.AUDIT.SCAN_VIOLATIONS` (latest scan_id)
- Row click → `st.expander` showing:
  - **WHY**: rationale text
  - **FIX**: remediation guidance (looked up from rule registry metadata or stored in violation)
  - **Quick Win** badge if remediation_key exists
  - **Blast radius**: Account-level vs object-specific
- Group-by toggle: flat list vs grouped by pillar/concept

**Page 3 — Trends:**
- Line chart: compliance_score over time (last 90 days from SCAN_METADATA)
- Per-pillar score lines (from pillar_scores VARIANT)
- Regression highlights: red markers where score dropped > 5 points scan-over-scan
- Scan comparison: two date-pickers → show violations added/removed between scans (LEFT ANTI JOIN on rule_id + resource_name)
- Summary table: all scans with date, score, grade, violation counts
